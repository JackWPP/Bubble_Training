### 二、 后续道路指引：如何冲击 5% 的提升？

在 16GB 显存限制和目前的框架下，要硬拔 5% 的绝对/相对 mAP50（达到 0.838 左右），你们必须跳出“同层微观结构修补”的舒适区，向**“物理分辨率重构”**、**“密集目标评价机制”**以及**“高级训练流形”**开刀。

建议按以下优先级（从高到低）推进接下来的研究方向：

#### 优先道路 1：物理层级的特化重构 —— “弃 P5，保 P2” (针对你们的 Path B)
* **核心痛点：** 气泡太小，如果在 P3 的特征图上只剩不到 $2 \times 2$ 的像素点，再优秀的 P4 到 P3 的跨层融合也变不出花来。必须引入 P2 层（4倍下采样）。
* **打破 16GB 显存限制的绝招：** 既然你们的任务是“单类别微小气泡检测”，画面中大概率不存在占据半个屏幕的“超大型目标”。因此，**YOLO11 默认的 P5 层（32倍下采样）对你们而言完全是算力和参数的冗余浪费**。
* **具体操作：** 不要盲目增加 P2 导致显存溢出。直接修改模型的 YAML 配置文件，**彻底砍掉 P5 层的 Neck 和 Head 分支，将释放出的海量内存全部用于构建一个轻量级的 P2 检测头**（即架构变为 P2-P3-P4）。这在论文中是一个极其强有力的 Story：“摒弃大目标冗余，针对微小气泡特化定制的高分辨率感受野架构”。

#### 优先道路 2：直击密集目标的软肋 —— 重构标签分配与后处理 (针对你们的 Path E)
* **核心痛点：** 你们目前“刻意避免了 Assignment 和 NMS 的改变”，但这极大概率是卡住涨点的命门。对于密集重叠的小气泡，YOLO 默认的 `TaskAlignedAssigner` 强依赖真实框与预测框的 IoU。微小目标的预测框只要偏移 1 个像素，IoU 就会呈悬崖式暴跌，导致极其严重的分配冲突（相邻气泡互相干扰、好样本被误判为负样本）。
* **具体操作：** 既然架构红利见底，限制必须放开。
  1. **将 NWD 或高级 IoU 融入分配器 (Assigner)：** 既然已经证明 NWD Loss 有效，请务必将 NWD（或近年专为微小目标设计的 Inner-IoU、Shape-IoU）直接写进 Assigner 的匹配代价矩阵（Cost Matrix）中。让模型分配正负样本时不要纯靠经典 IoU。这往往能大幅拉升小目标的召回率。
  2. **替换后处理 NMS：** 极其密集的气泡在标准的 NMS 下，会被当作同一个目标而被误杀（暴击 Recall）。在推理阶段换用 **Soft-NMS** 或 **DIoU-NMS**，不增加任何训练成本，很可能直接白嫖 1~2 个百分点的 mAP50。

#### 优先道路 3：驯服高潜力的训练配方 (针对你们的 Path C)
文档提到 `COCO-init` 曾有一次跑到过 `0.8071`。这释放了一个强信号：**模型当前的参数容量绝对有能力突破 0.80 的大关**，只是因为自然图像和单类低频纹理差异太大，导致微调时发生了严重的梯度震荡或灾难性遗忘。
* **具体操作（稳定优化 Recipe）：**
  1. **渐进式冻结 (Freeze & Unfreeze)：** 前 10-15 个 Epoch **绝对冻结 Backbone**，只用较大的学习率训练你们新加入的模块和分类头。等新模块适应了气泡域后，再解冻全局用极小的学习率微调。这能极大保护 COCO 优秀的底层特征提取能力。
  2. **关闭末期的 Mosaic 数据增强：** YOLO 默认的 Mosaic 会把图片随机切分拼接。对于原本就微小的气泡，一切两半后就变成了无法辨认的“脏噪声”。**强烈建议在最后 15-20 个 Epoch 彻底关闭 Mosaic**（可替换为 Copy-Paste 或尺度缩放）。
  3. **启用 SWA (随机权重平均)：** 面对高方差表现，SWA 是平滑优化空间陡峭程度、锁定高分的最佳武器。

#### 优先道路 4：跨尺度注入的防混叠设计 (针对你们准备执行的 Path A)
如果为了充实论文结构，必须执行 `P4TOPDOWN` 的跨尺度反馈：
* **避坑指南：** 气泡边缘本身就模糊，如果将 P4 简单粗暴地上采样（如 Nearest 插值）然后与 P3 相加，会带来严重的**锯齿混叠效应 (Aliasing)**，这反而会破坏 P3 辛辛苦苦提取的细粒度边界。
* **建议玩法：** 采用**上下文引导 (Context-Guided)** 思路。利用 P4 广阔的感受野生成一个**空间权重掩码 (Spatial Attention Mask)**，去乘上（Multiply）P3 的特征，用于抑制 P3 浅层特征中的背景假阳性（如水体反光），最后再通过你们的 Zero-initialized 残差缩放标量加进去。
