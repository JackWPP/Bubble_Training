# 05 — 讨论与失败分析

## 概述

本章汇集实验中的负结果分析、种子稳定性研究和增强管道饱和讨论。在学术写作中，"不 work 的尝试"与"work 的尝试"具有同等价值——前者记录了探索过程，也为社区提供了可复现的经验。

---

## 一、负结果分析

以下 6 个方向在实验中经过系统验证，未带来正向收益。每项分析包含假设、测试结果和根本原因分析。

### 1.1 WIoU v3 损失 (−0.018)

**假设**：加权 IoU（Wise-IoU v3）通过动态非单调聚焦机制，根据离群度自适应地调整不同质量锚框的梯度权重，应能提升小目标的训练稳定性。

**实验配置**：将 CIoU 损失完全替换为 WIoU v3，其余配置保持不变。

**结果**：综合指标 S 从 1.23882 降至 1.22108（−0.01774）。性能退化严重。

**根因分析**：
- WIoU 的离群度 β 通过 EMA 追踪 running mean of LIoU 实现，在小批量（batch=8）下估计不稳定
- COCO 预训练权重使用 CIoU 优化，切换损失函数使预训练权重中的 IoU 相关的 head 参数完全偏离最优区域
- 对于单类检测任务，WIoU 的动态聚焦机制引入的额外复杂度未带来等价的收益

**经验**：更换预训练模型的损失函数时需谨慎——COCO 预训练权重的 head 部分与 CIoU 损失深度耦合，替换需配合 head 重新初始化或极低学习率的渐进式适应。

### 1.2 AdamW 优化器 (−0.015)

**假设**：AdamW 的自适应学习率能更好地处理不同尺度的梯度，可能优于 SGD 在小数据集上的表现。

**实验配置**：将 SGD+Momentum 替换为 AdamW (lr=0.0005, weight_decay=0.0005)，其余不变。

**结果**：综合指标 S 从 1.25554 降至 1.24039（−0.01515）。

**根因分析**：
- SGD+Momentum 的隐式正则化（通过恒定学习率+动量产生的噪声）在小数据集上是优势——它防止模型过早陷入尖锐局部最优
- AdamW 的二阶矩估计在小批量下可能高估梯度方向的一致性，导致过早收敛
- 小数据集上的泛化能力对优化器选择更为敏感

**经验**：对于基于预训练模型的小数据集微调场景，SGD+Momentum 仍然是更安全的选择。

### 1.3 测试时增强 TTA (−0.019)

**假设**：测试时对输入图像进行多次增强（翻转+缩放），融合多次推理结果，应能提升检测精度。

**实验配置**：Ultralytics 内置 TTA（水平翻转 + 多尺度推理）。

**结果**：主测试集 mAP@50 从 0.81712 降至 0.79807（−0.01905）。精确率从 0.797 降至 0.779。

**根因分析**：
- TTA 在密集小目标场景中引入了假阳性（精确率↓）：多次推理在密集区域产生更多高置信度预测，NMS 融合时假阳性累积
- 对于单类检测，TTA 的"投票"机制（多尺度多翻转结果综合）不如多类检测有效——因为没有类别间的"共识"信号

**经验**：TTA 对单类密集小目标检测可能不适用，假阳性风险高于真阳性增益。

### 1.4 随机旋转增强 (−0.004)

**假设**：随机旋转（±5°）增强模型对气泡旋转的鲁棒性。

**结果**：综合指标 S 从 1.25554 降至 1.25143（−0.00411）。

**根因分析**：
- 气泡在显微图像中的空间排列（如电流场方向排列、浓度梯度方向排列）是有物理意义的信息，随机旋转破坏了这种排列模式
- 气泡近似圆形，旋转不变性本身已有一定基础，额外的旋转增强信息增量极小

**经验**：数据增强需要考虑目标在场景中的物理先验——对于有自然排列模式的目标，破坏空间结构的增强可能有害。

### 1.5 长训练周期 E50 (−0.004)

**假设**：50 轮训练能让模型更充分地拟合数据。

**结果**：综合指标 S 从 1.25554 降至 1.25039（−0.00515），且在约第 18 轮达到最优后性能持续下降。

**根因分析**：
- 644 张训练图的小数据集在约 14-18 轮后开始过拟合（通过 train-dev split 上的 mAP 趋势确认）
- 余弦学习率调度在 30 轮后学习率已接近零，额外 20 轮几乎没有有效学习

**经验**：小数据集上的最优训练轮数需要通过验证集趋势确定，并非越长越好。

### 1.6 NWD 归一化常数 C=6.4 (−0.020)

**假设**：更小的 C 值使 NWD 对位置偏移更敏感，增强定位精度。

**结果**：综合指标 S 从 1.26300 降至 1.24273（−0.02027）。

**根因分析**：
- C=6.4 时 NWD 函数过于陡峭，小偏移产生大梯度，与 CIoU 的梯度方向形成冲突
- 对于气泡尺寸约 41-42px 的目标，配 C=6.4 的 NWD 对标记者间约 2-3px 的正常标注偏差产生过度惩罚
- C=12.8 在这一目标尺寸下恰好提供了一个"宽容区"

**经验**：NWD 的常数 C 与目标尺度强相关——目标越小，C 应越大（更宽容）；目标越大，C 可更小。

## 二、种子稳定性分析

### 2.1 旧配方（无 Mosaic 增强）下的种子敏感度

在 P3LCRefine + NWD 但无 Mosaic 增强的条件下，测试 3 个随机种子（42, 44, 48）。

| 种子 | 综合指标 S |
|------|----------|
| 42 | 1.22885 |
| 44 | 1.23962 |
| 48 | 1.22148 |

- 均值: 1.22998
- 标准差: 0.00907
- 变化范围: 0.01814

### 2.2 新配方（Mosaic + HSV + MixUp）下的种子敏感度

| 种子 | 综合指标 S |
|------|----------|
| 44 | 1.25554 |
| 48 | 1.26300 |

- 均值: 1.25927
- 变化范围: 0.00746

### 2.3 分析

新配方不仅提升了绝对性能，还将种子间的变化范围从 0.01814 缩减至 0.00746（缩小约 2.4 倍）。Mosaic 增强通过每轮构建不同的拼接组合，增加了训练数据的有效多样性，降低了对特定初始化和数据顺序的敏感度。

值得注意的是，种子 48 在旧配方下表现最差（1.22148），在新配方下表现最优（1.26300）。这表明强增强配方使不同随机初始化的模型都能更好地收敛到高质量解，而非仅"利好"特定种子。

**论文建议**：
- 以种子 44 为主结果（实验全程默认种子，无选择偏差）
- 以种子 48 为补充结果（展示模型的性能上限）
- 若增加种子 42 和 49 的实验（推荐但非必需），可报告均值±标准差

## 三、增强管道饱和分析

### 3.1 饱和现象

在 Paper V4 数据集（644 张训练图）上，增强管道的边际收益呈现递减：

| 增强步骤 | 增益 | 说明 |
|---------|------|------|
| 无增强 → Mosaic(0.5)+HSV | +0.01179 | 大幅提升 |
| Mosaic(0.5)→(1.0) | +0.00307 | 已减弱 |
| +MixUp | +0.00106 | 边际收益 |
| +CutMix | ±0.000 | 零增益 |
| 所有进一步尝试 | ≤0 | 饱和 |

饱和的原因为**数据多样性已接近上限**：644 张原始图像来自约 170 个不同的物理采集场景。Mosaic 通过拼接组合创造了数量巨大的合成样本（任意 4 张的组合），有效地覆盖了训练所需的视觉多样性。此时增加增强方法的类型和强度不再引入新的信息维度。

### 3.2 突破饱和的可能方向

以下方向可能在当前基础上进一步突破（但未在本文中实施）：
- **SAHI 切片推理**（预期 +5-7%）：在推理时对大图进行滑动窗口切片后分别检测，再融合结果，可提升小目标的召回率
- **增加训练数据**：扩充训练集到 2000+ 张，引入更多采集条件
- **P2 高分辨率检测头**：增加 stride=4 的检测头处理更小目标
- **知识蒸馏**：用 YOLO11m 教师模型进行特征级蒸馏

## 四、方法论反思

### 4.1 成功经验

1. **控制变量法**：每次实验只改变一个因素，确保收益可归因
2. **双阈值权重筛选**：精度≥0.75 和召回率≥0.72 的筛选条件防止了极端但脆弱的最优解
3. **Train-dev split 隔离**：模型选择与最终评估严格分离，避免测试信息泄漏
4. **负结果的记录**：记录了 6 个不 work 的方向，为后续研究节约了重复探索成本

### 4.2 局限性

1. **仅测试 2 个种子**：新配方仅运行了种子 44 和 48，统计学意义有限
2. **单数据集验证**：仅在气泡检测任务上验证了改进效果，未测试其他小目标检测任务（如细胞检测、微小缺陷检测）的泛化性
3. **无与外部方法的直接对比**：仅与内部基线对比，未引入如 HSMD-YOLO、PC-YOLO 等同期小目标改进方法的复现对比
4. **OOD 性能小幅退化**：改进模型在 OOD 测试集上的 mAP@50 略低于基线，虽然 mAP@50:95 的退化更小
5. **标注质量未量化**：未评估标注者间一致性（inter-annotator IoU），无法确定 mAP 是否已达到标注水平的上限
