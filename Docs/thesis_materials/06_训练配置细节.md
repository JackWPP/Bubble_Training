## 二、权重选择策略

### 2.1 在线筛选模式（训练过程中）

训练过程中每个 epoch 在 train-dev split 的开发集上评估 mAP@50。权重筛选流程：

1. 记录每个 epoch 在开发集上的精确率、召回率、mAP@50、mAP@50:95
2. 过滤：精确率 ≥ 0.75 **且** 召回率 ≥ 0.72
3. 在通过过滤的 epoch 中，选取 mAP@50 最高者
4. 若无 epoch 通过过滤（极端情况），回退至全局最优 mAP@50 的 epoch
5. 选中的权重保存为 `weights/map50_selected.pt`

### 2.2 双阈值过滤的动机

在单类气泡检测中，mAP@50 可能被以下极端情况虚高：
- 高精确率低召回率：模型过于保守，仅检测置信度极高的气泡，但漏检大量真实气泡
- 高召回率低精确率：模型过于激进，产生大量假阳性

双阈值（P≥0.75, R≥0.72）确保选出的模型在精确率和召回率之间达到合理的平衡，综合性能更鲁棒。

### 2.3 Train-dev split

| 属性 | 值 |
|------|-----|
| 来源 | yolo_dataset_grouped 的 train split (3000 张) |
| 拆分比例 | 85% train / 15% dev |
| 最少 dev 样本数 | 50 |
| 拆分约束 | source_key 隔离（同一来源的样本全部在同侧） |
| 验证脚本 | `tools/validate_train_dev_split.py` |
| 配置文件 | `configs/data/bubble_train_dev.yaml` |

## 五、实验管理

### 5.1 实验命名规范

实验代号格式：`PV4C1_P3LC_COCO_NWD_W005_MOSAIC10_HSV_MIXUP_E30_seed48`

| 字段 | 含义 |
|------|------|
| PV4 | Paper V4 数据集 |
| C1 | COCO 预训练初始化 |
| P3LC | P3LCRefine 模块 |
| NWD_W005 | NWD 损失融合权重 0.05 |
| MOSAIC10 | Mosaic 概率 1.0 |
| HSV | HSV 色彩扰动 |
| MIXUP | MixUp 增强 |
| E30 | 30 训练轮数 |
| seed48 | 随机种子 48 |

### 5.2 结果存储

每个实验在 `runs/bubble_paper_v4_coco_aug/<experiment_name>/` 下存储：

- `weights/best.pt` —— Ultralytics 默认 fitness 最优
- `weights/last.pt` —— 最后一轮权重
- `weights/map50_selected.pt` —— 经双阈值筛选的最优权重（**论文使用**）
- `results.csv` —— 逐 epoch 的训练/验证指标
- `results.png` —— 训练曲线图
- `summary.json` —— 完整实验摘要
- `*.png` —— 精确率/召回率/F1 曲线、混淆矩阵等

所有实验的汇总 CSV 位于 `runs/summary/paper_v4_all_experiment_summary_latest.csv`。

### 5.3 复现说明

要复现论文结果：
1. 构建 Paper V4 数据集：`python 07_build_integrated_dataset.py --split paper_v4`
2. 运行训练：`python scripts/train_experiment.py --experiment PV4C1_P3LC_COCO_NWD_W005_MOSAIC10_HSV_MIXUP_E30_seed48`
3. 或直接加载已训练的 `map50_selected.pt` 权重进行评估

训练入口脚本：`scripts/train_experiment.py`
批量运行脚本：`scripts/run_nightly.py`
