# 01 — SAM3 零样本分割标注管线

## 概述

本章描述如何利用 SAM3 (Segment Anything Model 3) 将 COCO bbox-only 标注升级为高质量的实例分割多边形标注。SAM3 是 Meta 于 2025 年发布的统一视觉基础模型，支持以边界框为提示词进行零样本实例分割。

## 一、标注管线架构

```
COCO bbox-only → SAM3 推理 → COCO bbox+polygon → build_dataset.py → YOLO-seg 数据集
```

### 1.1 输入数据

7 个 COCO bbox-only 数据源，来自不同实验条件和成像设备：

| 数据源 | 图片数 | 标注框数 | 典型图像尺寸 |
|--------|--------|---------|-------------|
| 20+40 | 66 | 2,233 | 502×404 |
| 60+80 | 47 | 442 | 各种 |
| big_fengchao | 6 | 1,460 | 1920×1080 |
| bubble_1 | 3 | 590 | ~2450×? |
| bubble_fc | 12 | 602 | 各种 |
| bubble_pad | 3 | 1,484 | ~2560×? |
| job_13 | 33 | 2,713 | 各种 |
| **合计** | **170** | **9,524** | |

### 1.2 SAM3 模型

- 权重: `/home/xgx/sam3/` (model.safetensors, 3.4GB)
- 来源: HuggingFace `facebook/sam3` (gated model)
- 架构: Sam3VideoModel (ViT backbone 32层, CLIP text encoder, DETR detector, mask decoder)
- 处理器: `Sam3Processor` (image size 1008×1008, mask size 288×288)

### 1.3 多阶段推理策略

由于 SAM3 的注意力复杂度为 O(n²)，直接用全部 bbox 作为提示词会导致显存溢出。采用四阶段策略：

**Phase 1 — 概念分割（Concept Segmentation）**
- 随机采样最多 20 个 bbox 作为 exemplars
- 一次性输入 SAM3，触发概念学习
- 输出初步掩码

**Phase 2 — IoU 匹配**
- 将 SAM3 输出掩码与所有标注 bbox 进行 IoU 匹配
- 阈值: IoU ≥ 0.3
- 仅保留匹配置信度 ≥ 0.3 的掩码

**Phase 3 — 批量回退（Batch Fallback）**
- 未匹配的 bbox 按 20 个一组输入 SAM3
- 每组独立推理 + IoU 匹配

**Phase 4 — 单框回退（Single-box Fallback）**
- 仍然未匹配的 bbox 逐个输入 SAM3
- 取最佳 IoU 匹配或最高分掩码

### 1.4 后处理

- Polygon 简化: `cv2.approxPolyDP` (epsilon=1.0px)
- 最大顶点数: 50
- 最小面积: 4px²
- 输出格式: COCO 标准 `[[x1, y1, x2, y2, ..., xn, yn]]`

## 二、推理结果

| 数据源 | 总标注 | 有分割 | 覆盖率 |
|--------|--------|--------|--------|
| 20+40 | 2,233 | 2,233 | 100% |
| 60+80 | 442 | 442 | 100% |
| big_fengchao | 1,460 | 1,451 | 99.4% |
| bubble_1 | 590 | 589 | 99.8% |
| bubble_fc | 602 | 602 | 100% |
| bubble_pad | 1,484 | 1,475 | 99.4% |
| job_13 | 2,713 | 2,712 | 99.96% |
| **合计** | **9,524** | **9,504** | **99.79%** |

仅 20 个标注 (0.21%) 无法生成分割掩码，均为极小或边缘气泡。

## 三、运行环境

- GPU: 2× Tesla V100-SXM2-16GB (16GB VRAM)
- 显存消耗: 约 9-13GB (取决于图像尺寸和 bbox 数量)
- 推理速度: ~2-5s/图 (V100)
- 服务器 RAM: 15GB 限制, 需 `low_cpu_mem_usage=True`

## 四、输出格式

```json
{
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "bbox": [22.91, 358.87, 11.29, 10.23],
      "segmentation": [[24.0, 359.0, 23.0, 360.0, 22.0, 366.0, ...]]
    }
  ]
}
```

多边形格式为 COCO 标准: `[[x1, y1, x2, y2, ..., xn, yn]]` (绝对像素坐标)。

## 六、局限性

1. Polygon 最多 50 个顶点, 可能损失边界细节
2. 瓦片化处理 (stride=480) 在边界切割气泡
3. SAM3 概念模式在大 bbox 数量时召回率偏低
