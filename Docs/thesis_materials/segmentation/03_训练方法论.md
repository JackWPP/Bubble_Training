# 03 — 分割训练方法论

## 概述

本章描述 Bubble-YOLO11s-seg 的模型架构、训练策略和核心改进——Dice Loss 混合损失函数。经过 17 个消融实验系统验证，最终方案为 YOLO11s-seg + SGD + cos_lr + 零增强 + Dice Loss (w=0.1)。

## 一、基础模型: YOLO11s-seg

### 1.1 架构

YOLO11s-seg 是 Ultralytics 官方分割模型，包含：

- **Backbone**: YOLO11s (Conv + C3k2 + SPPF + C2PSA), 10 层, 约 10M 参数
- **Neck**: FPN 风格双向特征金字塔, P3/P4/P5 三尺度输出
- **Head**: Segment head, 32 个原型掩码 (Proto masks), 256 隐藏通道
- **FLOPs**: ~33G @ 640×640

### 1.2 预训练

使用 COCO 实例分割预训练权重 `yolo11s-seg.pt`。该权重在 COCO-seg 数据集上以标准配置训练完成，提供了强大的基础特征提取能力。

### 1.3 分割原理

YOLO 分割采用原型掩码 + 系数向量的线性组合机制：

```
Mask = Σ_i (coeff_i × proto_i), i = 1..32
```

其中 `coeff` 为每个检测框的 32 维掩码系数向量，`proto` 为全图级 32 通道原型掩码图 (160×160 @ 640px)。

## 二、训练策略演进

### 2.1 Baseline: AdamW + 中等增强

初始配置沿用 Ultralytics 默认推荐：

| 参数 | 值 |
|------|-----|
| 优化器 | AdamW |
| 学习率 | 0.001 |
| 学习率调度 | 线性衰减 (lrf=0.01) |
| 增强 | mosaic=0.3, mixup=0.1, HSV |
| Batch | 4 @ 640×640 |

结果: Mask mAP50=0.836, mAP50-95=0.529, best@epoch 46。

### 2.2 SGD + cos_lr + 零增强 (来自检测 pipeline 启发)

深度分析 72 个检测实验后，发现检测最佳配置与此完全不同：

| 参数 | 检测最佳 | 分割默认 |
|------|---------|---------|
| 优化器 | **SGD** | AdamW |
| cos_lr | **True** | False |
| 增强 | **mosaic=0** | mosaic=0.3 |

迁移到分割后: Mask mAP50-95=0.528 (几乎持平), 收敛速度提升 **4.6×** (epoch 10 vs 46)。

### 2.3 架构改进尝试 (全部失败)

项目自带 16 个自定义 PyTorch 模块 (来自检测消融), 逐一测试：

| 模块 | 原理 | 效果 |
|------|------|------|
| P3LCRefine | 局部对比度增强 (高通滤波) | mAP50↑, mAP50-95↓ |
| P3MLCRefine | 多尺度对比度 (3×3+5×5) | 持平 |
| P3SAGate | 空间注意力门控 | mAP50-95↓ |
| P3+P4+P5 LCRefine | 三尺度全部放置 | mAP50-95↓ |
| SPD-Conv | 无损降采样 | mAP50-95↓ |
| Proto=64 | 双倍原型掩码 | mAP50-95↓ |

**结论**: 检测特征增强模块在分割上系统性失效——强化平移不变性特征会破坏分割所需的平移可变性空间细节。

## 三、核心改进: Dice Loss 混合损失

### 3.1 动机

BCE Loss 将掩码内每个像素视为独立同分布，在全局平均下，少量边界像素的误差被大量内部像素稀释。Dice Loss 直接优化预测与真实掩码的重叠度 (IoU)，对边界误差施加更强梯度：

$$\text{Dice} = \frac{2|P \cap G| + 1}{|P| + |G| + 1}, \quad \mathcal{L}_{\text{Dice}} = 1 - \text{Dice}$$

### 3.2 混合方案

将 Dice Loss 与 BCE Loss 加权混合，通过 monkey-patch `v8SegmentationLoss.single_mask_loss` 实现：

$$\mathcal{L}_{\text{mask}} = (1-w) \cdot \mathcal{L}_{\text{BCE}} + w \cdot \mathcal{L}_{\text{Dice}}$$

### 3.3 Dice Weight 消融

| Dice weight w | Mask mAP50 | Mask mAP50-95 | 评价 |
|:---:|-----------|--------------|------|
| 0 (纯 BCE) | 0.829 | 0.528 | baseline SGD |
| 0.05 | 0.829 | 0.520 | 太弱, 无效 |
| **0.10** | **0.839** | **0.538** | **最优** ★ |
| 0.30 | 0.828 | 0.532 | 次优 |
| 0.50 | 0.826 | 0.520 | 过强, 退化 |

**最优值 w=0.1**: Dice 提供边界梯度增强, BCE 维持训练稳定性。过高的 Dice 权重会导致梯度振荡。

### 3.4 实现

```python
def _patched_single_mask_loss(gt_mask, pred, proto, xyxy, area):
    pred_mask = torch.einsum("in,nhw->ihw", pred, proto)
    bce = F.binary_cross_entropy_with_logits(pred_mask, gt_mask, reduction="none")
    
    pred_prob = pred_mask.sigmoid()
    intersection = (pred_prob * gt_mask).sum(dim=(1, 2))
    union = pred_prob.sum(dim=(1, 2)) + gt_mask.sum(dim=(1, 2))
    dice = (2.0 * intersection + 1.0) / (union + 1.0)
    dice_loss = 1.0 - dice
    
    bce_cropped = crop_mask(bce, xyxy).mean(dim=(1, 2)) / area
    return ((1 - 0.1) * bce_cropped + 0.1 * dice_loss).sum()
```

## 四、为何其他方法失败

### 4.1 NWD Loss

NWD 将 bbox 建模为 2D 高斯分布, 改善小目标召回。但在分割任务中, NWD 始终以牺牲 mAP50-95 为代价提升 mAP50——表明它提升了检测置信度但损害了定位精度。

### 4.2 架构改动

16 个自定义模块在检测上有效 (+0.01-0.02 mAP50) 但在分割上无效。原因是这些模块设计用于增强分类鲁棒性 (平移不变性), 而分割需要精确的空间定位 (平移可变性)。

### 4.3 高分辨率 (768px)

V100 16GB 显存限制下 batch=2 太小, BatchNorm 统计不稳定, 抵消了分辨率收益。梯度累积 (nbs=4) 部分缓解但未完全解决。
