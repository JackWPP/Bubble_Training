# 02 — 分割数据集构建

## 概述

本章描述如何将 SAM3 增强的 COCO JSON 标注转换为 YOLO-seg 格式的训练数据集。数据集构建遵循与检测数据集一致的 Paper V4 分层分割策略，采用 640×640 瓦片化处理以适应模型输入。

## 一、数据集构建管线

```
SAM3 输出 (COCO bbox+polygon) → build_dataset.py → YOLO-seg 数据集
```

### 1.1 输入

- 7 个 COCO 数据源的 `instances_default_segmented.json`
- 每个源包含 `images/` 和 `annotations/` 两个子目录
- 总计 170 张原图, 9,504 个多边形标注

### 1.2 输出

```
segmentation/datasets/paper_v4_seg/
├── bubble_seg.yaml
├── train/
│   ├── images/    (712 张 640×640 瓦片)
│   └── labels/    (712 个 .txt 标签文件)
├── val/
│   ├── images/    (63 张)
│   └── labels/
└── test/
    ├── images/    (69 张)
    └── labels/
```

## 二、数据集分割策略

### 2.1 分层分割

- **分层单位**: `source_key = "{source_name}_{file_name}"`
- **分割比例**: 70% train / 15% val / 15% test
- **随机种子**: 44 (确定性可复现)

### 2.2 分割统计

| 分割 | 原图数 | 瓦片数 | 标签文件数 |
|------|--------|--------|-----------|
| train | 118 | 356 | 712 (含 hflip) |
| val | 25 | 63 | 63 |
| test | 27 | 69 | 69 |
| **合计** | **170** | **488** | **844** |

### 2.3 增强

仅训练集应用水平翻转 (hflip) 增强，将训练瓦片数翻倍至 712 张。

## 三、瓦片化处理

### 3.1 大图瓦片化 (>640px)

对最长边超过 640px 的图像进行滑动窗口瓦片化：
- **瓦片尺寸**: 640×640
- **步长**: 480 (重叠 160px)
- **多边形裁剪**: 使用 Shapely 将多边形裁剪到瓦片边界
- **质量过滤**: 裁剪后面积 < 原面积 40% 的标注被丢弃

### 3.2 小图 Letterbox (<640px)

对最长边小于 640px 的图像进行 letterbox 填充：
- 保持宽高比缩放
- 灰色填充至 640×640

### 3.3 空瓦片处理

最多保留 10% 的空瓦片（无标注）作为负样本。

## 四、标签格式

### 4.1 YOLO-seg 标签格式

每个 `.txt` 文件包含多行，每行一个实例：

```
<class_id> <x1> <y1> <x2> <y2> ... <xn> <yn>
```

- class_id: 固定为 0 (单类 "bubble")
- 坐标: 归一化到 [0, 1] (相对于 640×640 瓦片)

### 4.2 数据集配置文件

`bubble_seg.yaml`:
```yaml
path: /path/to/paper_v4_seg
train: train/images
val: val/images
test: test/images
names:
  0: bubble
nc: 1
```

## 五、与检测数据集的对比

| 属性 | 分割数据集 | 检测数据集 |
|------|-----------|-----------|
| 原始图片 | 170 张 (相同) | 170 张 |
| 标注类型 | Polygon (SAM3) | BBox (人工) |
| 训练瓦片 | 356 (+356 hflip) | 644 (不同分割) |
| 验证瓦片 | 63 | 83 |
| 测试瓦片 | 69 | 83 |
| 分割比例 | 70/15/15 | 不同比例 |

分割数据集与检测数据集使用相同的原始图片，但分割策略不同。分割数据集有更多训练瓦片（因为 SAM3 polygon 标注覆盖了更多区域），但验证/测试集更小。

## 六、数据集局限性

1. 瓦片化在边界处切割气泡，可能引入标注噪声
2. 验证集仅 63 张，mAP 指标方差较大
3. SAM3 生成的 polygon 最多 50 个顶点，可能损失边界细节
4. 训练集仅 170 张原始图片，对分割任务可能不够充分
