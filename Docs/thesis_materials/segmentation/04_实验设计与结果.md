# 04 — 分割实验设计与结果

## 概述

本章报告 Bubble-YOLO11s-seg 的完整实验过程。共进行 **17 个实验**，分 6 轮递进：架构改进 → 训练策略 → gamma 调优 → SGD 优化器 → Dice Loss → 最终调优。实验遵循统一的评估标准：COCO Mask mAP@50 和 Mask mAP@50:95。

## 一、实验设计

### 1.1 实验环境

| 组件 | 配置 |
|------|------|
| GPU | Tesla V100-SXM2-16GB (单卡) |
| CUDA | 13.0 |
| PyTorch | 2.4.0+cu124 |
| ultralytics | 8.3.227 |
| 预训练 | yolo11s-seg.pt (COCO 分割) |

### 1.2 评估指标

采用 COCO 标准 mAP 指标（单类情形下等价于 AP）：
- **Mask mAP50**: 粗粒度掩码检测精度 (IoU threshold=0.50)
- **Mask mAP50-95**: 细粒度掩码定位精度 (IoU threshold=0.50:0.05:0.95)

### 1.3 基线配置

| 参数 | 值 |
|------|-----|
| 模型 | YOLO11s-seg (标准架构) |
| 优化器 | AdamW |
| 学习率 | 0.001 |
| Batch | 4 @ 640×640 |
| 增强 | mosaic=0.3, mixup=0.1, HSV |
| Epochs | 100 (early stop patience=30) |

基线结果: **Mask mAP50=0.836, Mask mAP50-95=0.529, best@epoch 46**。

## 二、消融实验

### 2.1 Round 1: 架构改进 (5 实验)

目标: 测试项目中 16 个自定义模块在分割任务上的效果。

| 实验 | 配置 | mAP50 | mAP50-95 | ΔmAP50-95 |
|------|------|-----------|--------------|-----------|
| ① Baseline | YOLO11s-seg | 0.836 | 0.529 | — |
| ② + P3LCRefine + NWD | 局部对比度 + NWD | 0.847 | 0.508 | -0.021 |
| ③ + P3MLCRefine | 多尺度对比度 | 0.836 | 0.519 | -0.010 |
| ④ + P3LCRefine + P3SAGate | 对比度 + 空间注意力 | 0.838 | 0.501 | -0.028 |
| ⑤ + P3+P4+P5 LCRefine | 三尺度对比度 | 0.840 | 0.513 | -0.016 |

**结论**: 所有架构改动均导致 mAP50-95 下降。P3LCRefine+NWD 虽然 mAP50 最高 (0.847)，但定位精度受损。

### 2.2 Round 2: 训练策略 (2 实验)

| 实验 | 配置 | mAP50 | mAP50-95 |
|------|------|-----------|--------------|
| ⑥ Baseline + NWD | w_nwd=0.05 | 0.842 | 0.509 |
| ⑦ Baseline + 768px | batch=2 | 0.838 | 0.521 |

**结论**: NWD 提升 mAP50 但损害 mAP50-95。768px 因 batch=2 太小未体现收益。

### 2.3 Round 3: gamma 调优 (2 实验)

| 实验 | gamma | mAP50 | mAP50-95 |
|------|:---:|-----------|--------------|
| ⑧ P3LCRefine g=0.01 | 0.01 | 0.835 | 0.513 |
| ⑨ P3LCRefine g=0.1 | 0.1 | 0.830 | 0.512 |

**结论**: 降低 gamma_init 不能恢复 COCO 特征，P3LCRefine 收益来自即时高通滤波。

### 2.4 Round 4: SGD 优化器迁移 (3 实验)

深度分析 72 个检测实验后，将检测最佳配置迁移到分割：

| 实验 | 配置 | mAP50 | mAP50-95 | Best Epoch |
|------|------|-----------|--------------|:---:|
| ⑩ SGD + 零增强 | SGD, cos_lr, mosaic=0 | 0.829 | **0.528** | **10** |
| ⑪ SGD + 当前增强 | SGD, cos_lr, mosaic=0.3 | 0.822 | 0.519 | 44 |
| ⑫ 检测 BB 迁移 | det best.pt → seg | 0.834 | 0.499 | 26 |

**关键发现**: SGD + 零增强 (⑩) 以 4.6× 更快收敛几乎追平 baseline mAP50-95，验证了检测 pipeline 的有效性。

### 2.5 Round 5: Dice Loss 核心突破 (4 实验)

| 实验 | Dice w | mAP50 | mAP50-95 | Best Epoch |
|------|:---:|-----------|--------------|:---:|
| ⑬ Dice w=0.3 | 0.3 | 0.828 | 0.532 | 13 |
| ⑭ **Dice w=0.1** ★ | **0.1** | **0.839** | **0.538** | **12** |
| ⑮ Dice w=0.5 | 0.5 | 0.826 | 0.520 | 22 |
| ⑯ Dice w=0.05 | 0.05 | 0.829 | 0.520 | 8 |

**关键发现**: Dice w=0.1 实现了 mAP50 (+0.003) 和 mAP50-95 (+0.009) **双提升**，收敛仅需 12 epoch。

### 2.6 Round 6: 最终调优 (3 实验)

| 实验 | 配置 | mAP50 | mAP50-95 |
|------|------|-----------|--------------|
| ⑰ Dice 0.1 + NWD | combo | 0.829 | 0.526 |
| ⑱ Dice 0.1 + AdamW | 原始增强 | 0.835 | 0.522 |
| ⑲ 768px + Dice 0.1 + nbs=4 | 高分辨率 | 0.827 | 0.532 |

**结论**: Dice w=0.1 + SGD + 零增强 仍是最优组合。NWD、AdamW、768px 均无法进一步改善。

## 三、最优模型

**配置**: YOLO11s-seg, SGD, cos_lr, 零增强, Dice Loss (w=0.1), 640×640, batch=4

| 指标 | Baseline | 最优模型 | 提升 |
|------|---------|---------|------|
| Mask mAP50 | 0.836 | **0.839** | +0.003 |
| Mask mAP50-95 | 0.529 | **0.538** | +0.009 |
| Box mAP50 | 0.845 | 0.837 | -0.008 |
| Box mAP50-95 | 0.532 | 0.542 | +0.010 |
| 收敛速度 | epoch 46 | **epoch 12** | 3.8× faster |

**模型路径**: `segmentation/runs/bubble_seg/bubble_seg_dice01/weights/best.pt`

## 四、改进历程

```
Baseline (mAP50-95=0.529)
  → SGD+cos_lr+零增强 (0.528, 收敛 4.6× fast)
  → Dice Loss w=0.1 (0.538 ★, 同时提升 mAP50)
```

仅通过**训练策略优化**（优化器 + 学习率调度 + 损失函数），在不改变模型架构、不增加参数的前提下，将 mAP50-95 从 0.529 提升至 0.538 (+1.7%)，同时 mAP50 从 0.836 提升至 0.839 (+0.4%)。
