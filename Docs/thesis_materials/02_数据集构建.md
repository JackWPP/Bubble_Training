# 02 — 数据集构建

## 概述

气泡检测任务的数据集经历了三代迭代，从最初的随机拆分综合数据集到最终的场景隔离论文数据集。每一代针对前一版本暴露的问题进行了改进。

---

## 一、原始数据来源

原始数据来自 7 个不同物理采集条件下的 COCO 格式标注数据集，所有标注类别统一映射为单类 `bubble`（类别编号 0）。

| 来源 | 原始图像数 | 有效标注框数 | 典型分辨率 | 采集特征 |
|------|----------|------------|-----------|---------|
| 20+40 | 66 | 2,233 | 502×400 | 显微镜静态拍摄，气泡密集 |
| 60+80 | 47 | 442 | 508×425 | 显微镜静态拍摄，气泡稀疏 |
| big_fengchao | 6 | 1,460 | 1920×1080 | 视频帧，大视场，高密度 |
| bubble_1 | 3 | 590 | 1920×1080 | 视频帧，特定工况 |
| bubble_fc | 12 | 602 | 480×480 | 视频帧，多电流参数条件 |
| bubble_pad | 3 | 1,484 | 1920×1080 | 视频帧，特定采集板 |
| job_13 | 33 | 2,713 | 1920×1080 | 视频帧，多浓度参数条件 |

**总计**：170 张原始图像，9,524 个标注框。原始标注类别仅 `bubble` 一类。

数据来源的多样性涵盖了不同采集设备（显微镜/摄像头）、分辨率（480×480 至 1920×1080）、密度（稀疏至密集）和成像条件（电流、浓度参数变化），为模型泛化能力提供了基础。

## 二、数据集版本演进

### 2.1 V1：综合数据集 (yolo_dataset_integrated)

**目标**：快速构建可用训练集，验证训练流程可行性。

**拆分策略**：以原始图像为基本单位，按 80/15/5 比例随机拆分至 train/val/test。随机种子固定为 42。

**图像处理**：
- 任一边大于 640px：使用 640×640 滑动窗口切片，水平与垂直 stride 均为 480px，边缘自动补最后一窗
- 任一边不大于 640px：保持比例缩放，居中填充至 640×640
- 标注框保留条件：（1）中心点落入切片内；（2）裁剪后宽高均 ≥ 4px；（3）保留面积 ≥ 原面积的 40%
- 空切片（无标注框）最多保留正样本切片数的 10%

**离线增强**（仅作用于 train 集）：
- 水平/垂直翻转
- 90°、180°、270° 旋转
- 轻度亮度、对比度、HSV 调整
- 轻度高斯噪声
- Copy-Paste 增强（气泡裁剪仅来自 train 集，避免泄漏）

**输出统计**：

| Split | 图像数 | 标注框数 | 平均框/图 |
|-------|--------|---------|----------|
| train | 5,715 | 176,145 | 30.82 |
| val | 84 | 1,908 | 22.71 |
| test | 23 | 515 | 22.39 |

**已知局限**：
- 随机原图拆分：来自同一视频的相邻帧可能分别进入 train 和 val/test，无法评估跨场景泛化能力
- 部分来源（bubble_pad、bubble_1）未进入 val/test，来源覆盖不完整
- 离线增强后的 val/test 不经过增强，但 train 集中包含同一物理帧经过离线增强的变体

**定位**：早期开发集，用于快速验证训练流程和消融架构假设。不适合作为论文正式测试集。

### 2.2 V2：按源分组数据集 (yolo_dataset_grouped)

**目标**：解决 V1 的场景泄漏问题，构建严格的分布外（Out-of-Distribution, OOD）泛化测试基准。

**拆分策略**：先按物理源头和采集条件生成 `group_key`（如 `big_fengchao|video_or_scene`、`bubble_fc|-40-0.16A-1(480P).mp4`），再按 group_key 做 train/val/test 划分。目标比例 75/10/15。

**核心原则**：同一物理采集条件（同一视频、同一实验批次）的**所有图像**必须分配到同一 split 中。任何 group_key 不能跨 train/val/test 出现。

**图像处理**：与 V1 相同（640×640 滑窗，stride 480）。

**离线增强**（仅作用于 train 集）：水平/垂直翻转、90°/180°/270° 旋转、轻度亮度/对比度调整。

**输出统计**：

| Split | 图像数 | 标注框数 | 平均框/图 | 离线增强样本 |
|-------|--------|---------|----------|------------|
| train | 3,000 | 56,445 | 18.82 | 200 基础 → 2,800 |
| val | 80 | 4,195 | 52.44 | 0 |
| test | 208 | 6,108 | 29.37 | 0 |

**泄漏验证**：
- 精确原图 `source_key` 跨 split：**0**
- 物理源头 `group_key` 跨 split：**0**
- val/test 中离线增强样本：**0**

**来源覆盖**：

| 来源 | train | val | test | 说明 |
|------|-------|-----|------|------|
| 20+40 | 990 | 0 | 0 | 仅训练 |
| 60+80 | 705 | 0 | 0 | 仅训练 |
| big_fengchao | 0 | 0 | 48 | 仅测试（OOD） |
| bubble_1 | 0 | 0 | 24 | 仅测试（OOD） |
| bubble_fc | 105 | 8 | 0 | 训练+验证 |
| bubble_pad | 0 | 24 | 0 | 仅验证 |
| job_13 | 1,200 | 48 | 136 | 全部覆盖 |

**核心设计意图**：故意将 `20+40`、`60+80` 全部放入训练集，将 `big_fengchao`、`bubble_1` 全部放入测试集。这意味着测试集中的采集条件在训练中完全不可见，是严格的分布外泛化测试。若模型仅在训练分布上过拟合，OOD 测试集上的性能会显著下降。

**定位**：论文正式分布外泛化测试集。既可用于验证模型是否真正学到了气泡的本质特征（而非特定采集条件的伪影），也可评估数据增强对跨域泛化的贡献。

### 2.3 V3：论文数据集 Paper V4 (yolo_dataset_paper_v4)

**目标**：在 V2 基础上，使用 768×768 分辨率，仅保留在线增强，构建论文的主实验数据集。

**构建参数**：
- 输出目录：`yolo_dataset_paper_v4`
- 拆分模式：`paper_v4`
- 图像尺寸：768×768（直接缩放，不使用滑动窗口）
- 训练集/验证集/测试集：70% / 15% / 15%（按 source_key 隔离拆分）
- 离线增强：仅水平翻转（fliplr）

**与 V2 的关键差异**：

| 维度 | V2 (Grouped) | V3 (Paper V4) |
|------|-------------|---------------|
| 分辨率 | 640×640（滑窗） | 768×768（直接缩放） |
| 训练集大小 | 3,000 | 644 |
| 测试集大小 | 208 | 83 |
| 拆分方式 | group_key 隔离 | source_key 隔离 |
| 离线增强程度 | 翻转+旋转+亮度 | 仅水平翻转 |
| 在线增强 | 无 | **Mosaic+HSV+MixUp**（核心改进） |

**Paper V4 的输出统计**：

| Split | 图像数 |
|-------|--------|
| train | 644 |
| val | 83 |
| test | 83 |

**为什么选择 768×768 而非 640×640？**

气泡作为小目标，在 1920×1080 原图中尺寸约为 30-80px。若直接缩放至 640，气泡将被压缩至约 10-27px，损失大量纹理信息。768 分辨率下，气泡平均尺寸约 47px，在 P3 特征层（stride=8）上对应约 6×6 个 grid cell，保留了足够的局部结构信息用于检测和定位。

**为什么 Paper V4 不使用滑动窗口？**

滑动窗口（640×640, stride 480）将大图切分为多个子图，会增加训练集数量（3,000），但也带来了两个问题：
1. 同一张原图的多个切片在特征上高度相关，增大了训练集规模但没有等比例增加信息多样性
2. 切片可能将完整的气泡截断，引入边界伪影

Paper V4 刻意使用 644 张直接缩放图像的小数据集，目的是在**数据稀缺**的设定下测试模型和增强配方的鲁棒性——更贴近实际应用中标注数据有限的场景。

## 三、训练-开发集分离机制

### 3.1 动机

在模型训练过程中，需要从多个 epoch 的权重中选择最优模型。如果使用测试集进行选择，会导致测试信息泄漏（"间接过拟合"）。标准的做法是将训练集进一步拆分为训练子集和开发集（dev set）。

### 3.2 实现

从 V2 数据集的 train split（3,000 张）中，按 `source_key` 隔离拆分出约 15% 作为开发集（最少 50 张），随机种子 42。

约束条件：
- 仅非增强的基础样本（`raw`、`letterbox`、`tile_*` 前缀）可被选入开发集
- 同一 `source_key` 的所有样本必须全部进入训练子集或全部进入开发子集
- 验证通过 `tools/validate_train_dev_split.py` 确认零泄漏

### 3.3 权重筛选协议

权重选择采用**双阈值过滤 + 最大 mAP@50** 策略：

1. 在 train-dev split 的开发集上评估每个 epoch 的权重
2. 过滤：精确率 (Precision) ≥ 0.75，召回率 (Recall) ≥ 0.72
3. 在通过过滤的 epoch 中选取 mAP@50 最高者
4. 若没有 epoch 通过过滤，回退到全局最优 mAP@50

选中的权重保存为 `map50_selected.pt`，后续在主测试集和 OOD 测试集上进行最终评估。

**为什么需要双阈值过滤？**

mAP@50 在单类检测中可能被异常高置信度的假阳性或假阴性影响。精确率和召回率的最低阈值确保选出的模型既不过于激进（低精度），也不过于保守（低召回），综合性能更稳健。

## 四、气泡尺寸分布

构建后（V1，640×640 滑窗后）的边界框尺寸统计：

| 统计量 | 宽度 (px) | 高度 (px) |
|--------|----------|----------|
| 均值 | 45.35 | 44.70 |
| 中位数 (p50) | 41.60 | 40.55 |
| p90 | 75.69 | 74.32 |
| 最小值 | 5.87 | 5.87 |
| 最大值 | 315.0 | 312.4 |

气泡目标的中位尺寸约 41-42px。按照 COCO 数据集的小目标定义（面积 ≤ 32×32 = 1024px²），约 40% 的气泡属于小目标。这解释了为何 P3 特征层（stride=8，最高分辨率）的改进最为关键。

## 五、数据集划分完整性验证

所有数据集版本均通过自动化脚本验证，检查项包括：

1. **跨 split 泄漏**：同一 `source_key` 或 `group_key` 不得出现在多个 split 中
2. **离线增强泄漏**：val 和 test 中不得包含离线增强样本
3. **标签格式**：YOLO 格式（5 列，归一化坐标 ∈ [0,1]，bbox 宽高 > 0）
4. **文件配对**：每张图像必须有同名标签文件

所有三个版本均通过全部检查项，验证结果为 **PASS**。
